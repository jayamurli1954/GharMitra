
# SOFTWARE REQUIREMENT SPECIFICATION  
System: GharMitra  
Module: Finance Governance & Reporting  
Derived from: 6-01-10-011  

---

## 1. Purpose
Ensure legally compliant financial visibility, privacy, and auditability in GharMitra by enforcing role-based access and structured finance reporting.

---

## 2. Scope
This SRS covers:
- Role-based access
- Member reports
- Committee finance UI
- Audit dashboards

It does not modify:
- Billing calculations
- Ledger posting logic

---

## 3. Roles

| Code | Role |
|------|------|
| SA | Super Admin |
| ADM | Society Admin |
| COM | Committee |
| MEM | Member |
| SEC | Security |
| AUD | Auditor |

---

## 4. Functional Requirements

### FR-1 Role Assignment  
Each user must have exactly one role per society.

### FR-2 Member Data Isolation  
Members may only access records tied to their flat_id.

### FR-3 Committee Finance Access  
ADM and COM must access all society financial data.

### FR-4 Audit Read Access  
AUD may read all financial data but cannot modify.

### FR-5 UI Enforcement  
UI must hide disallowed reports.

---

## 5. Non-Functional Requirements

- All data access must be validated server-side  
- All financial reads must be logged  
- All changes must be auditable  

---

## 6. Data Rules

MEM -> WHERE flat_id = user.flat_id
ADM/COM/AUD -> WHERE society_id = user.society_id
